# HomeLab AgentOps v0.5 - Public Toolkit Foundation

HomeLab AgentOps v0.5 turns this repository into a public-safe toolkit that can be tested without private HomeLab access.

## Purpose

This release provides a sanitized foundation for HomeLab diagnostics, public security checks, safe status reporting, shell validation, and backup dry-run workflows.

The goal is not to expose a private HomeLab.

The goal is to demonstrate safe operational patterns.

## Included Capabilities

- Public status check
- Backup dry-run workflow
- Strict public security scan
- Shell syntax validation
- Doctor check
- Public-safe demo output
- Contribution guidelines
- Security policy
- Public/private boundary documentation

## Public Commands

Run these commands from the repository root:

    bash scripts/halo-status.sh
    bash scripts/halo-backup-dryrun.sh
    bash scripts/halo-security-scan.sh --strict
    bash scripts/halo-doctor.sh

## Expected Results

Expected public-safe results:

- `HALO_PUBLIC_STATUS=GREEN` when no local warnings are detected.
- `HALO_PUBLIC_STATUS=WARN` when a local condition requires review.

    HALO_BACKUP_DRYRUN=OK
    Security scan result: GREEN
    Shell validation passed

## Safety Principles

The public toolkit must not require or expose:

- real NAS credentials
- Telegram bot tokens
- chat IDs
- private IP addresses
- SSH keys
- real backup destinations
- internal production logs
- private screenshots
- production-only runtime files

## What v0.5 Means

v0.5 is the first public toolkit foundation release.

It is no longer only documentation. It includes safe scripts, validation workflows, demo output, and security-first operating patterns.

## Future Direction

This repository remains focused on HomeLab operations:

- monitoring
- diagnostics
- backup discipline
- recovery documentation
- public-safe automation
- future local AI infrastructure readiness

Future local AI work should remain separate from this repository.

## Public and Private Boundary

The public repository is sanitized by design.

Private infrastructure details, secrets, production-only paths, real backup archives, and private logs must stay outside this repository.
